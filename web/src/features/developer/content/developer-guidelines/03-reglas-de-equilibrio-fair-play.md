# ⚖️ Reglas de equilibrio (Fair Play)

Para garantizar una competición justa, todos los juegos deben cumplir **reglas mínimas de equilibrio** y mecanismos de prevención frente a ventajas injustas.

> [!IMPORTANT]
> El objetivo de estas reglas es proteger la **integridad competitiva** y asegurar que el resultado de una partida dependa principalmente del desempeño de los jugadores.

## Igualdad de condiciones

Todos los jugadores deben iniciar la partida con las **mismas condiciones y recursos**.

## Ausencia de ventajas injustas

No se permitirán mecánicas que otorguen **ventajas arbitrarias** a un jugador.

## Control del azar

El uso de elementos aleatorios debe ser **moderado** y estar siempre **controlado**, evitando que el resultado de la partida dependa excesivamente de la suerte.

| Uso del azar     | Criterio                             |
| ---------------- | ------------------------------------ |
| **Permitido**    | Añadir variabilidad.                 |
| **Permitido**    | Generar dinamismo.                   |
| **No permitido** | Determinar directamente el ganador.  |
| **No permitido** | Invalidar la estrategia del jugador. |

> [!WARNING]
> Un sistema donde la suerte tenga más peso que la habilidad puede romper el equilibrio competitivo y afectar negativamente a la experiencia de juego.

## 🛠️ Validación en servidor

Las acciones críticas deben validarse en el servidor para evitar trampas.

> [!IMPORTANT]
> Las decisiones relevantes para el resultado de la partida **no deben depender únicamente del cliente**.

## ⚠️ Prevención de exploits

El diseño del juego debe anticipar posibles abusos o usos indebidos del sistema.

Se deben evitar:

* Estrategias que rompan el juego.
* Bucles infinitos o mecánicas abusivas.
* Acciones que permitan evitar reglas.
